---
name: follow-root-structure
description: Enforce current root directory classification for new work
type: feedback
---
Always create/move new work to match the current root folder classification (00-Inbox, 01-Assets, 02-Docs, 03-Scripts, 04-Configs, 05-Outputs, 06-Archive, 99-Projects-Other, bo-work, bo-skill-research, etc.).
**Why:** User wants strict adherence to the newly organized root structure for all future Claude Code actions.
**How to apply:** When creating files/projects or moving outputs, place them into the appropriate existing bucket instead of adding new top-level folders or leaving items at root.
