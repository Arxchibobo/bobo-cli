# 高级工程模式（源自 Claude Code 架构）

## 记忆类型学（四类型）

记忆不是什么都存的垃圾桶。只存**不能从当前项目状态推导出来**的信息。

### 存什么
| 类型 | 说明 | 示例 |
|------|------|------|
| user | 用户的角色、目标、偏好、知识水平 | "用户是高级后端工程师，第一次碰 React" |
| feedback | 用户给的行为指导（纠正 + 确认） | "不要在这些测试里 mock 数据库" |
| project | 进行中的工作、目标、截止日期 | "Q2 目标是把延迟降到 200ms 以下" |
| reference | 不在代码中的外部知识 | "生产数据库在 us-east-1 的 RDS 上" |

### 不存什么（NOT-to-save）
- ❌ 代码模式/架构（grep 能查到）
- ❌ Git 历史（git log 能看到）
- ❌ 文件结构（ls/find 能列出）
- ❌ 已在 CLAUDE.md/AGENTS.md 中的规则
- ❌ 调试中间步骤（只存最终教训）
- ❌ 一次性信息（临时状态）

### feedback 记忆格式
```
规则本身
**Why:** 用户为什么这么说（事故/偏好/上下文）
**How to apply:** 在什么场景下应用
```

**关键**：不只记纠正，也记确认。只记纠正 = 过度谨慎 + 偏离已验证的方法。

## Compact 九段式（详细版）

对话过长时的压缩输出结构：

1. **主要请求和意图** — 用户所有明确请求的详细描述
2. **关键技术概念** — 讨论过的所有重要技术/框架/概念
3. **文件和代码** — 检查/修改/创建的具体文件和代码段，包含完整代码片段
4. **错误和修复** — 遇到的所有错误及修复方式，特别是用户反馈
5. **问题解决** — 已解决的问题和进行中的排错
6. **所有用户消息** — 列出所有非工具结果的用户消息（**关键！理解用户反馈和变化的意图**）
7. **待办任务** — 明确被要求做的待办事项
8. **当前工作** — 摘要请求前正在做什么（含文件名和代码片段）
9. **下一步** — 与最近工作直接相关的下一步，附原文引用防止意图漂移

## 背景记忆提取模式

每轮对话结束后自动检查是否有值得记录的信息：
1. 第一轮：并行读取所有可能需要更新的记忆文件
2. 第二轮：并行写入所有更新
3. 不要交错读写 — 效率优先

## 子代理架构模式

### 内置代理类型
- **Explore Agent** — 轻量搜索模式，最少 token 完成调研
- **Verification Agent** — 对抗性验证，尝试打破而不是确认
- **Plan Agent** — 任务规划和分解

### Coordinator 模式
大任务的四阶段：
1. **研究**（并行）— 多个子代理同时调研不同方面
2. **综合**（自己）— 汇总研究结果
3. **实现**（可并行）— 分配子任务
4. **验证**（独立）— 独立代理验证结果

### 并发安全
- 扇出并行 / 扇入汇总
- 读写分离（多读串写）
- 依赖串行（有依赖的步骤不并行）
