# 常见错误速查

## Top 高频错误

| ID | 错误 | 核心问题 | 快速修复 |
|----|------|---------|---------|
| E001 | 异步未并行 | 多个独立异步操作串行执行 | `Promise.all()` |
| E002 | 轮询无超时 | 无限循环等待 | 设 `maxAttempts` + backoff |
| E003 | 错误未抛出 | catch 块吞掉错误 | catch 块 `throw` 或 log + re-throw |
| E004 | SQL 未用 CTE | 大表全扫 | 预过滤 + CTE |
| E005 | 路径硬编码 | 跨平台失败 | `path.join()` / `path.resolve()` |
| E006 | 编码假设 | 非 UTF-8 文件乱码 | 检测编码，指定 encoding |
| E007 | 资源泄漏 | 文件句柄/连接未关闭 | 所有退出路径都 cleanup |
| E008 | 竞态条件 | 并发读写同一文件 | 锁 / 串行化 |
| E009 | 默认分页 | API 只取了第一页 | 循环分页拉完再统计 |
| E010 | rebase 丢代码 | 静默丢失改动 | push 前 `git diff HEAD~1` |

## 调试 Checklist

1. ✅ 拿到确切错误信息（不靠猜）
2. ✅ 确认复现步骤
3. ✅ 定位到具体文件和行号
4. ✅ 理解根因（不只是症状）
5. ✅ 最小改动修复
6. ✅ 验证修复有效
7. ✅ 检查无回归

## Shell 命令安全

- 危险命令（rm -rf, DROP TABLE 等）**必须**先确认路径/范围
- 生产环境操作用 `--dry-run` 先预览
- 超过 30s 的命令设 timeout
- 大量输出用 `| head -n 50` 截断
