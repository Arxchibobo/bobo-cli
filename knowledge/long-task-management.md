# 长任务管理协议

## 判定标准
预计 ≥30 分钟的异步任务，必须走此协议。

## 启动清单

### 1. 设置心跳监控
```
启动后 10 分钟内必须设心跳：
- 进程活着 + 有进展 + 资源健康
- 完成后自动执行后续
- 失败自愈（3 次失败才找人）
```

### 2. 三文件模式
```
task_plan.md  — 规划 + 进度追踪
notes.md      — 过程笔记 + 发现
[deliverable] — 最终交付物
```

### 3. Plans.md 表格（6+ 步任务）
```markdown
| Task | 内容 | DoD（Yes/No） | Depends | Status |
|------|------|-------------|---------|--------|
| T1   | ...  | ...         | -       | TODO   |
```
- Status: TODO → WIP → Done → Verified
- Depends=`-` 可并行
- 循环依赖 → 报错重规划

## 进度汇报

### 里程碑快照格式
```
📊 Progress: Task X/Y 完成
状态: [在做/卡住/等决策]
已完成: [列表]
进展: [当前在干什么]
ETA: [预估完成时间]
需要: [如果需要对方做什么]
```

### 卡点上报时机
- 重复失败 ≥3
- 流程阻塞（等人/等资源）
- 信息缺失
- 工具/环境故障

## Cron 心跳规范

### timeout 设置
- 纯脚本/curl: 60-300s
- 涉及 LLM 多轮对话: **900s（15min）**
- 涉及视频生成/训练: 1800s+

### 自愈策略
```
失败 1 次 → 重试（不同策略）
失败 2 次 → 重试（换方案）
失败 3 次 → 上报 + 暂停
```

### 完成后自动后续
- 下载产出
- 跑验证
- 通知相关方
- 更新状态

## 资源监控

启动大任务前检查：
- `df -h /` — 磁盘空间（<10% 就先清理）
- `free -m` — 内存
- 网络连通性
- API quota/余额

遇到 `No space left on device` → **1小时内同步卡点**，不自己死磕。
