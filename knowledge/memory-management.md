# 记忆管理系统

## 架构

### 文件结构
| 文件 | 用途 | 上限 |
|------|------|------|
| MEMORY.md | 热记忆（每 session 自动加载） | ≤15KB |
| memory/YYYY-MM-DD.md | 每日日志 | 不限 |
| memory/archive/ | 归档冷数据 | 不限 |

### MEMORY.md 固定结构
```markdown
## 🔒 用户偏好（永久）
## 💼 角色定位（永久）
## 📝 业务经验（≤15 条）
## 📋 活跃任务（完成即删）
## 🔄 纠正记录（≤10 条）
## 💬 对话摘要（7 天后精简归档）
```

## 主动写入（7 个触发条件）

不等用户说"记住"：
1. 用户纠正 → 纠正记录
2. ≥3 轮才搞清 → 业务经验
3. 用户做决策 → 对话摘要
4. ≥5 步复杂任务 → 业务经验
5. 数据口径/技巧 → 业务经验
6. 用户偏好 → 用户偏好
7. 任务搁置 → 活跃任务

## 纠正记录格式
```markdown
### [YYYY-MM-DD] 简短标题
- 我原来以为: xxx
- 正确理解: xxx
- **Why:** 根因
- **How to apply:** 适用场景
- 来源: 主人纠正
```

## 纠正判断（宽松识别）
任何用户回应表明之前理解/做法/结论有误：
- 直接否定 / 给出不同答案 / "其实..."
- 示范不同做法 / 质疑 / 放弃让你做
- 第三方指出错误

## 晋升规则
- 重复 ≥3 次 → 晋升到 AGENTS.md（硬规则）
- 对其他人有价值 → 共享知识库
- 高频经验 → 从 MEMORY 提升到 knowledge 文件

## 漂移防护
- 记忆中的路径 → 使用前 check exists
- 记忆中的函数名 → grep 确认
- 记忆与当前事实矛盾 → 以当前状态为准 + 更新记忆
- compact 后 → 九段式保留关键上下文

## 归档策略
- 7 天以上对话摘要 → 精简归档
- 完成的活跃任务 → 删除
- 纠正记录 > 10 条 → 归档最早的
- 业务经验 > 15 条 → 归档低频的
- 月度：`du -sh archive/` > 500MB → 清物理大文件

## Session 恢复（跨 session 断片）
容器重启/session 重置后：
1. 检查最近 session 文件
2. 检查 `.reset.*` 后缀文件
3. grep 关键词恢复前序进展
4. 读 MEMORY.md 活跃任务 section
