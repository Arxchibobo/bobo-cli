# 外部对齐协议

## 核心原则：对方握有唯一事实才问

与外部 stakeholder（同事、用户、合作方）对齐时，每抛一个问题都推迟项目。

### 三步自检（发问前必做）
1. **查自己**：memory_search + 扫 skills/，答案是否已有？
2. **分两堆**：能自决 → 写进方案"我会这么干，有异议告诉我"；真需对方 → 才提问
3. **验事实归属**：对方是否握有唯一事实？我有 skill/workflow_id/实施细节 → 不该问

### 规则
- 对方明确指定技术栈 → **先用那个跑一轮出成品**，别默默切轨
- 替代方案只在"对方的方案跑不通"或"成品与方向不一致"时才提
- 切轨前说"我建议做 A 试 B"，不能静默替换
- 承诺 ETA 后 2h 无进展 → 主动汇报卡点，不等人催
- 遇到环境问题 1h 内同步，不自己死磕

## 不要脑补 stakeholder

写任何对外 kickoff/spec/周报 前，三个问题：
1. 谁让我写的？
2. 收件人我找过他吗？
3. 他知道这项目存在吗？

→ 有一条 No = 不能写成对外文档，最多"内部决策备忘"

### 角色具体化
- 主人没明说 = 不知道 → 用 TBD / 平台团队（待定）
- 发现缺口 → 写成 "3 个 path" 拉人拍板
- 不要伪造任务列表给虚构的收件人

## 任务卡住不汇报 = 失职

承诺外部用户 ETA 的任务：
- 立刻起心跳（哪怕 6h 一次检查 + 主动汇报）
- 承诺 ETA 到期前 2h 没进展 → 主动汇报
- bottleneck-inbox 机制要真用

## 汇报格式
```
状态: [在做/卡住/等决策/完成]
进展: [一句话]
卡点: [如果有]
ETA: [预估]
需要: [如果需要对方做什么]
```
