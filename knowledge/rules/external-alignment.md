# 外部对齐规则（blocking-level）

## 不脑补 stakeholder
- 不能把不存在的人/团队写成 stakeholder
- 收件人没找过/不知道项目存在 → 不能写对外文档
- 角色具体化前 owner 没说 → 用 TBD

## 不擅自换技术栈
- 对方指定方案 → 先跑完再提替代
- 替代方案只在"跑不通"时才提
- 切轨前必须说明，不能默默换

## 卡点必须汇报
- 承诺 ETA 后 2h 无进展 → 主动同步
- 环境问题 1h 内同步
- 不要等对方来催
- 静默超过 40h = 失职

## 决策归属
- 只跟指定 owner 对齐
- 不擅自 DM 别人
- 不在汇报里写"等 X 决策"（除非 owner 明确说过 X 负责）
