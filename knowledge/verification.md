# 验证协议

## 何时必须验证

以下情况完成后**必须**执行验证：
- ≥3 个文件被编辑
- 后端 / API 变更
- 基础设施 / 配置变更
- 任何"应该能工作"的直觉判断

## 验证方法

### 基础验证（必做）
1. **编译检查**：代码改完能编译通过
2. **测试运行**：有测试的跑测试
3. **手动验证**：没测试的手动执行一遍
4. **输出检查**：确认输出符合预期

### 对抗性验证（推荐）
至少一个对抗性探测：
- **边界值**：空输入、超大输入、特殊字符
- **幂等性**：同一操作执行两次，结果是否一致
- **错误恢复**：中途失败能否正确回滚/清理
- **并发安全**：多个请求同时执行

## 验证输出格式

```
✅ PASS: [what was verified]
❌ FAIL: [what failed + error details]
⚠️ SKIP: [what wasn't verified + reason]
```

## 反模式（不要做）
- ❌ 只读代码不运行就说"看起来对"
- ❌ 假装跑了测试
- ❌ 成功路径验证通过就收工（要测失败路径）
- ❌ "应该没问题" — 不是验证
